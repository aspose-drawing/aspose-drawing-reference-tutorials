# Aspose.Drawing for .NET में कोऑर्डिनेट सिस्टम ट्रांसफ़ॉर्मेशन (पेज ट्रांसफ़ॉर्मेशन)

## परिचय

स्वागत है! इस ट्यूटोरियल में आप **एक पेज पर कोऑर्डिनेट सिस्टम ट्रांसफ़ॉर्मेशन** कैसे किया जाता है, यह सीखेंगे, Aspose.Drawing for .NET का उपयोग करके। चाहे आपको **रेखाचित्र बिटमैप** ऑब्जेक्ट्स बनाना हो, ड्रॉइंग को स्केल करना हो, या पेज यूनिट्स को डिवाइस यूनिट्स में मैप करना हो, नीचे दिए गए चरण आपको पूरी प्रक्रिया के माध्यम से मार्गदर्शन करेंगे—स्पष्ट, संक्षिप्त, और आपके प्रोजेक्ट में कॉपी‑पेस्ट करने के लिए तैयार।

## त्वरित उत्तर
- **ड्रॉइंग के लिए मुख्य क्लास कौन सी है?** Aspose.Drawing की `Graphics`।
- **मैं पेज यूनिट्स कैसे सेट करूँ?** `graphics.PageUnit = GraphicsUnit.Inch;` का उपयोग करें।
- **क्या मैं ट्रांसफ़ॉर्म्ड पेज पर एक आयत (rectangle) बना सकता हूँ?** हाँ—एक `Pen` बनाएं और `DrawRectangle` कॉल करें।
- **आउटपुट कहाँ सहेजा जाता है?** `bitmap.Save` में आप जो फ़ोल्डर निर्दिष्ट करेंगे, वहाँ।
- **क्या उत्पादन (production) के लिए लाइसेंस चाहिए?** व्यावसायिक उपयोग के लिए एक वैध Aspose.Drawing लाइसेंस आवश्यक है।

## कोऑर्डिनेट सिस्टम ट्रांसफ़ॉर्मेशन क्या है?

एक **कोऑर्डिनेट सिस्टम ट्रांसफ़ॉर्मेशन** लॉजिकल पेज कोऑर्डिनेट्स (जैसे इंच या मिलीमीटर) को फिजिकल डिवाइस कोऑर्डिनेट्स (पिक्सेल) में कैसे मैप किया जाता है, इसे बदलता है। ट्रांसफ़ॉर्मेशन को समायोजित करके, आप सभी ड्रॉइंग ऑपरेशन्स के स्केलिंग, रोटेशन और ट्रांसलेशन को नियंत्रित कर सकते हैं, जिससे रेज़ॉल्यूशन‑इंडिपेंडेंट ग्राफ़िक्स बनाना आसान हो जाता है।

## पेज ट्रांसफ़ॉर्मेशन के लिए Aspose.Drawing क्यों उपयोग करें?

- **पूर्ण .NET संगतता** – .NET Framework, .NET Core, और .NET 5/6 के साथ काम करता है।
- **समृद्ध ग्राफ़िक्स API** – System.Drawing.Common जैसी कार्यक्षमता प्रदान करता है, बिना प्लेटफ़ॉर्म प्रतिबंधों के।
- **सरल यूनिट हैंडलिंग** – एक ही प्रॉपर्टी से इंच, मिलीमीटर, पॉइंट आदि के बीच स्विच करें।
- **उच्च‑गुणवत्ता आउटपुट** – PNG, JPEG, BMP और अन्य फ़ॉर्मेट्स को सटीक नियंत्रण के साथ जेनरेट करें।

## पूर्वापेक्षाएँ

शुरू करने से पहले सुनिश्चित करें कि आपके पास है:

- **Aspose.Drawing लाइब्रेरी** – आधिकारिक साइट [यहाँ](https://releases.aspose.com/drawing/net/) से नवीनतम संस्करण डाउनलोड करें।
- **डेवलपमेंट एनवायरनमेंट** – Visual Studio (कोई भी एडिशन) या कोई अन्य .NET IDE।
- **राइट एक्सेस** – आपके मशीन पर वह फ़ोल्डर जहाँ ट्रांसफ़ॉर्म्ड इमेज़ सहेजी जाएगी (कोड में `"Your Document Directory"` को वास्तविक पाथ से बदलें)।

अब जब सब तैयार है, चलिए प्रत्येक चरण को देखते हैं।

## नेमस्पेसेस इम्पोर्ट करें

सबसे पहले, आवश्यक नेमस्पेस को स्कोप में लाएँ:

```csharp
using System.Drawing;
```

> *यह क्यों महत्वपूर्ण है*: `System.Drawing` में `Bitmap`, `Graphics`, `Pen` और कलर स्ट्रक्चर होते हैं, जिनका उपयोग हम पूरे ट्यूटोरियल में करेंगे।

## चरण 1: एक Bitmap बनाएं

एक खाली कैनवास बनाएं जो ट्रांसफ़ॉर्म्ड ड्रॉइंग को होस्ट करेगा। हम **1000 × 800 पिक्सेल** आकार और अल्फा ट्रांसपैरेंसी सपोर्ट करने वाला पिक्सेल फॉर्मेट निर्दिष्ट करते हैं।

```csharp
Bitmap bitmap = new Bitmap(1000, 800, System.Drawing.Imaging.PixelFormat.Format32bppPArgb);
```

> यह bitmap ड्रॉइंग सतह के रूप में कार्य करता है। चुना गया पिक्सेल फॉर्मेट (`Format32bppPArgb`) प्री‑मल्टिप्लाइड अल्फा के साथ उच्च‑गुणवत्ता रेंडरिंग सुनिश्चित करता है।

## चरण 2: एक Graphics ऑब्जेक्ट बनाएं

`Graphics` ऑब्जेक्ट ड्रॉइंग मेथड्स (लाइन, शैप, टेक्स्ट) प्रदान करता है। हम इसे अभी बनाए गए bitmap से प्राप्त करते हैं।

```csharp
Graphics graphics = Graphics.FromImage(bitmap);
```

> `Graphics` ऑब्जेक्ट सभी रेंडरिंग ऑपरेशन्स का कोर है; यह उन ट्रांसफ़ॉर्मेशन सेटिंग्स का सम्मान करता है जिन्हें हम आगे लागू करेंगे।

## चरण 3: कैनवास को साफ़ करें

किसी भी चीज़ को ड्रॉ करने से पहले, कैनवास को एक तटस्थ बैकग्राउंड रंग (इस उदाहरण में ग्रे) से साफ़ करें। यह चरण यह भी दर्शाता है कि पूरे bitmap को एक सॉलिड रंग से कैसे भरें।

```csharp
graphics.Clear(Color.FromKnownColor(KnownColor.Gray));
```

## चरण 4: ट्रांसफ़ॉर्मेशन सेट करें (पेज ट्रांसफ़ॉर्मेशन लागू करें)

यहाँ हम **पेज ट्रांसफ़ॉर्मेशन** लागू करते हैं, `PageUnit` को इंच पर सेट करके। यह ग्राफ़िक्स इंजन को बताता है कि सभी बाद के कोऑर्डिनेट्स पिक्सेल के बजाय इंच में व्याख्यायित हों।

```csharp
graphics.PageUnit = GraphicsUnit.Inch;
```

> *टिप*: `PageUnit` बदलना **पेज कोऑर्डिनेट्स को ट्रांसफ़ॉर्म** करने का एक सरल तरीका है, बिना पिक्सेल वैल्यूज़ को मैन्युअली गणना किए।

## चरण 5: एक आयत (Rectangle) बनाएं (draw rectangle bitmap)

अब हम एक आयत ड्रॉ करते हैं जिसका आकार **1 इंच × 1 इंच** है, स्थिति (1, 1) इंच पर। `Pen` की चौड़ाई `0.1f` इंच सेट की गई है, जिससे लाइन पतली लेकिन स्पष्ट दिखे।

```csharp
Pen pen = new Pen(Color.FromKnownColor(KnownColor.Blue), 0.1f);
graphics.DrawRectangle(pen, 1, 1, 1, 1);
```

> यह **draw rectangle bitmap** को ट्रांसफ़ॉर्मेशन लागू करने के बाद दर्शाता है—ध्यान दें कि आयत का आकार इंच में परिभाषित है, पिक्सेल में नहीं।

## चरण 6: इमेज़ को सहेजें

अंत में, ट्रांसफ़ॉर्म्ड bitmap को डिस्क पर सहेजें। `"Your Document Directory"` को आपके मशीन पर वास्तविक फ़ोल्डर पाथ से बदलें।

```csharp
bitmap.Save("Your Document Directory" + @"CoordinateSystemsTransformations\PageTransformation_out.png");
```

> आउटपुट फ़ाइल (`PageTransformation_out.png`) में वह आयत होगी जो हमने पहले सेट किए गए कोऑर्डिनेट सिस्टम ट्रांसफ़ॉर्मेशन का उपयोग करके ड्रॉ किया था।
