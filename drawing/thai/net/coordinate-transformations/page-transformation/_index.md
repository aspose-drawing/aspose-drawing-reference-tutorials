# การแปลงระบบพิกัด – การแปลงหน้าใน Aspose.Drawing สำหรับ .NET

## บทนำ

ยินดีต้อนรับ! ในบทแนะนำนี้คุณจะได้ค้นพบ **วิธีแปลงพิกัดของหน้า** ด้วย Aspose.Drawing สำหรับ .NET และเรียนรู้พื้นฐานของ **การแปลงระบบพิกัด** ไม่ว่าคุณจะกำลังสร้างแอปพลิเคชันที่ต้องการกราฟิกหนักหรือจำเป็นต้องควบคุมหน่วยการวาดอย่างแม่นยำ คู่มือนี้จะพาคุณผ่านทุกขั้นตอน—from การตั้งค่าแคนวาสจนถึงการวาดรูปสี่เหลี่ยม. เมื่อเสร็จสิ้น คุณจะสามารถนำเทคนิคเหล่านี้ไปใช้ในโปรเจกต์ของคุณได้อย่างมั่นใจ

## คำตอบอย่างรวดเร็ว
- **การแปลงระบบพิกัดคืออะไร?** การแมปหน่วยระดับหน้ากระดาษ (เช่น นิ้ว) ไปยังพิกเซลระดับอุปกรณ์  
- **ทำไมต้องใช้ Aspose.Drawing?** ให้ทางเลือกที่จัดการเต็มรูปแบบแทน System.Drawing.Common พร้อมการสนับสนุนข้ามแพลตฟอร์ม  
- **ใช้เวลานานเท่าไหร่ในการทำตัวอย่าง?** ประมาณ 5‑10 นาทีสำหรับการแปลงหน้าพื้นฐาน  
- **ต้องมีลิขสิทธิ์หรือไม่?** สามารถใช้รุ่นทดลองฟรีสำหรับการพัฒนา; ต้องมีลิขสิทธิ์เชิงพาณิชย์สำหรับการผลิต  
- **รองรับเวอร์ชัน .NET ใดบ้าง?** .NET Framework 4.6+, .NET Core 3.1+, .NET 5/6/7

## การแปลงระบบพิกัดคืออะไร?

**การแปลงระบบพิกัด** กำหนดวิธีที่หน่วยตรรกะของหน้า (เช่น นิ้ว, เซนติเมตร หรือพอยต์) ถูกแปลงเป็นพิกเซลของอุปกรณ์เมื่อเรนเดอร์กราฟิก โดยการกำหนดคุณสมบัติ `Graphics.PageUnit` คุณบอกเอนจินการวาดให้ตีความพิกัดที่คุณส่งเข้าไป ทำให้คุณควบคุมขนาดและการจัดวางได้อย่างละเอียด

## ทำไมต้องใช้การแปลงระบบพิกัดกับ Aspose.Drawing?

- **การออกแบบอิสระจากอุปกรณ์:** เขียนโค้ดครั้งเดียวแล้วให้ Aspose.Drawing จัดการสเกลพิกเซลสำหรับหน้าจอหรือเครื่องพิมพ์ใดก็ได้  
- **การวาดที่แม่นยำ:** เหมาะสำหรับแผนภาพเทคนิค, สเก็ตช์สไตล์ CAD, หรือสถานการณ์ใด ๆ ที่ต้องการการวัดที่แน่นอน  
- **ความน่าเชื่อถือข้ามแพลตฟอร์ม:** ทำงานสม่ำเสมอบน Windows, Linux, macOS โดยไม่มีข้อจำกัดของ GDI+ ใน System.Drawing

เมื่อทุกอย่างพร้อมแล้ว ไปสู่ขั้นตอนแบบละเอียดกันเถอะ

## นำเข้า Namespaces

เริ่มต้นโดยนำเข้า namespace ที่จำเป็นเข้าสู่โปรเจกต์ของคุณ:

```csharp
using System.Drawing;
```

## ขั้นตอนที่ 1: สร้าง Bitmap

เราจะเริ่มด้วยการสร้าง bitmap ว่างเปล่าที่ทำหน้าที่เป็นพื้นผิวการวาด รูปแบบพิกเซล `Format32bppPArgb` ให้คุณภาพสูงพร้อมการสนับสนุน alpha แบบ premultiplied

```csharp
Bitmap bitmap = new Bitmap(1000, 800, System.Drawing.Imaging.PixelFormat.Format32bppPArgb);
```

## ขั้นตอนที่ 2: สร้าง Graphics Object

อ็อบเจกต์ `Graphics` ให้ API การวาดสำหรับ bitmap ทำหน้าที่เป็นสะพานระหว่างโค้ดของคุณและบัฟเฟอร์พิกเซล

```csharp
Graphics graphics = Graphics.FromImage(bitmap);
```

## ขั้นตอนที่ 3: ล้างแคนวาส

ให้แคนวาสมีพื้นหลังเป็นสีกลางเพื่อให้รูปทรงที่วาดเด่นชัด ที่นี่เราจะเติมด้วยสีเทาอ่อน

```csharp
graphics.Clear(Color.FromKnownColor(KnownColor.Gray));
```

## ขั้นตอนที่ 4: ตั้งค่าการแปลง (วิธีแปลงหน้า)

เพื่อแมปพิกัดของหน้าไปยังพิกเซลของอุปกรณ์ ให้ตั้งค่าคุณสมบัติ `PageUnit` ในตัวอย่างนี้เราเลือกหน่วยเป็นนิ้ว แต่คุณก็สามารถใช้ `GraphicsUnit.Millimeter`, `Point` ฯลฯ ได้เช่นกัน

```csharp
graphics.PageUnit = GraphicsUnit.Inch;
```

## ขั้นตอนที่ 5: วาดสี่เหลี่ยม – draw rectangle graphics

ต่อไปเราจะวาดสี่เหลี่ยมโดยใช้ปากกาสีฟ้าแถบบาง เนื่องจากเราเปลี่ยนเป็นหน่วยนิ้ว ขนาดและตำแหน่งของสี่เหลี่ยมจึงถูกระบุเป็นนิ้ว ทำให้โค้ดอ่านง่ายสำหรับเลย์เอาต์ที่มุ่งเน้นการพิมพ์

```csharp
Pen pen = new Pen(Color.FromKnownColor(KnownColor.Blue), 0.1f);
graphics.DrawRectangle(pen, 1, 1, 1, 1);
```

## ขั้นตอนที่ 6: บันทึกภาพ

สุดท้ายให้เขียน bitmap ลงไฟล์ PNG ในโฟลเดอร์ที่คุณระบุไว้ก่อนหน้า

```csharp
bitmap.Save("Your Document Directory" + @"CoordinateSystemsTransformations\PageTransformation_out.png");
```

ยินดีด้วย! คุณเพิ่งทำ **การแปลงระบบพิกัด**, ตั้งค่า page unit เป็นนิ้ว, และ **draw rectangle graphics** บน bitmap ด้วย Aspose.Drawing

## ปัญหาที่พบบ่อยและวิธีแก้

| ปัญหา | สาเหตุ | วิธีแก้ |
|-------|--------|----------|
| **ไฟล์ผลลัพธ์ไม่ถูกสร้าง** | เส้นทางไม่ถูกต้องหรือโฟลเดอร์ไม่มีอยู่ | ตรวจสอบให้แน่ใจว่าไดเรกทอรีเป้าหมายมีอยู่หรือใช้ `Directory.CreateDirectory` ก่อนบันทึก |
| **สี่เหลี่ยมบิดเบี้ยว** | `PageUnit` ไม่ตรงหรือ DPI ไม่ตรงกัน | ยืนยันว่า `graphics.PageUnit` ตรงกับหน่วยที่คุณต้องการใช้และ DPI ของ bitmap ตั้งค่าอย่างเหมาะสม (ค่าเริ่มต้นคือ 96 DPI) |
| **ข้อยกเว้นลิขสิทธิ์** | รันโดยไม่มีลิขสิทธิ์ที่ถูกต้องในสภาพการผลิต | ใส่ลิขสิทธิ์ Aspose.Drawing ชั่วคราวหรือถาวรของคุณก่อนสร้างอ็อบเจกต์ graphics |
