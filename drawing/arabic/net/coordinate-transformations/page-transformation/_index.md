# تحويل نظام الإحداثيات – تحويل الصفحة في Aspose.Drawing لـ .NET

## مقدمة

مرحبًا! في هذا البرنامج التعليمي ستكتشف **كيفية تحويل إحداثيات الصفحة** باستخدام Aspose.Drawing لـ .NET وتتعلم أساسيات **تحويل نظام الإحداثيات**. سواءً كنت تبني تطبيقًا يعتمد على الرسومات بشكل مكثف أو تحتاج إلى تحكم دقيق في وحدات الرسم، فإن هذا الدليل سيرشدك خطوة بخطوة — من إعداد اللوحة إلى رسم عنصر رسومي على شكل مستطيل. في النهاية، ستكون قادرًا على تطبيق هذه التقنيات في مشاريعك بثقة.

## إجابات سريعة
- **ما هو تحويل نظام الإحداثيات؟** تحويل وحدات المستوى الصفحي (مثل البوصات) إلى بكسلات الجهاز.  
- **لماذا نستخدم Aspose.Drawing؟** لأنه يقدم بديلًا مُدارًا بالكامل لـ System.Drawing.Common مع دعم متعدد المنصات.  
- **كم من الوقت يستغرق تنفيذ المثال؟** حوالي 5‑10 دقائق لتحويل صفحة أساسي.  
- **هل أحتاج إلى ترخيص؟** النسخة التجريبية المجانية تكفي للتطوير؛ الترخيص التجاري مطلوب للإنتاج.  
- **ما إصدارات .NET المدعومة؟** .NET Framework 4.6+، .NET Core 3.1+، .NET 5/6/7.

## ما هو تحويل نظام الإحداثيات؟

**تحويل نظام الإحداثيات** يحدد كيفية تحويل الوحدات المنطقية للصفحة (مثل البوصات، السنتيمترات أو النقاط) إلى بكسلات الجهاز عند عرض الرسومات. من خلال ضبط خاصية `Graphics.PageUnit` تخبر محرك الرسم كيف يفسر الإحداثيات التي تزودها، مما يمنحك تحكمًا دقيقًا في الحجم والتخطيط.

## لماذا نستخدم تحويل نظام الإحداثيات مع Aspose.Drawing؟

- **تصميم غير معتمد على الجهاز:** اكتب الكود مرة واحدة ودع Aspose.Drawing يتولى تعديل البكسلات لأي شاشة أو طابعة.  
- **رسم دقيق:** مثالي للمخططات التقنية، الرسومات على نمط CAD، أو أي سيناريو يتطلب قياسات دقيقة.  
- **موثوقية متعددة المنصات:** يعمل بشكل متسق على Windows وLinux وmacOS دون قيود GDI+ في System.Drawing.

## استيراد المساحات الاسمية

أولًا، أضف مساحة الأسماء المطلوبة إلى مشروعك:

```csharp
using System.Drawing;
```

## الخطوة 1: إنشاء صورة نقطية (Bitmap)

نبدأ بإنشاء صورة نقطية فارغة ستعمل كسطح رسم. تنسيق البكسل `Format32bppPArgb` يمنحنا جودة عالية ودعم للشفافية المسبقة الضرب.

```csharp
Bitmap bitmap = new Bitmap(1000, 800, System.Drawing.Imaging.PixelFormat.Format32bppPArgb);
```

## الخطوة 2: إنشاء كائن Graphics

كائن `Graphics` يوفر واجهة برمجة تطبيقات الرسم للصورة النقطية. إنه الجسر بين الكود الخاص بك ومخزن البكسلات.

```csharp
Graphics graphics = Graphics.FromImage(bitmap);
```

## الخطوة 3: مسح اللوحة

امنح اللوحة خلفية محايدة حتى تبرز الأشكال المرسومة. هنا نقوم بملئها بلون رمادي فاتح.

```csharp
graphics.Clear(Color.FromKnownColor(KnownColor.Gray));
```

## الخطوة 4: ضبط التحويل (كيفية تحويل الصفحة)

لتحويل إحداثيات الصفحة إلى بكسلات الجهاز، اضبط خاصية `PageUnit`. في هذا المثال نختار البوصة، لكن يمكنك أيضًا استخدام `GraphicsUnit.Millimeter` أو `Point` وغيرها.

```csharp
graphics.PageUnit = GraphicsUnit.Inch;
```

## الخطوة 5: رسم مستطيل – draw rectangle graphics

الآن نرسم مستطيلًا باستخدام قلم أزرق رفيع. لأننا غيرنا الوحدة إلى بوصة، فإن حجم المستطيل وموقعه يُعبَّران بالبوصة، مما يجعل الكود أكثر قابلية للقراءة في التخطيطات الموجهة للطباعة.

```csharp
Pen pen = new Pen(Color.FromKnownColor(KnownColor.Blue), 0.1f);
graphics.DrawRectangle(pen, 1, 1, 1, 1);
```

## الخطوة 6: حفظ الصورة

أخيرًا، اكتب الصورة النقطية إلى ملف PNG في المجلد الذي حددته مسبقًا.

```csharp
bitmap.Save("Your Document Directory" + @"CoordinateSystemsTransformations\PageTransformation_out.png");
```

تهانينا! لقد نفذت **تحويل نظام الإحداثيات**، وضبطت وحدة الصفحة إلى بوصة، و**رسمت رسومات مستطيلة** على صورة نقطية باستخدام Aspose.Drawing.
